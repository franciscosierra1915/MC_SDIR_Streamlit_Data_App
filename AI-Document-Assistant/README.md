# AI Document Assistant (Streamlit RAG Mini-Lab)

## Overview
This project implements a simple AI-powered document assistant using **Streamlit** and the **Google Gemini API**.  
The application allows a user to upload a document (PDF or TXT) and ask questions about its contents.  

The system uses a Retrieval-Augmented Generation (RAG) style workflow where the AI model is instructed to answer questions **only using the uploaded document**.

This project was created as part of the **AI Integration Mini-Lab** assignment for the CIS 3590 course.

---

# Model Information

**Model Name:** Gemini 3 Flash Preview  
**Provider:** Google AI (Google GenAI API)

Gemini is a multimodal large language model capable of processing text and documents.  
The **Gemini Flash model** was selected because it is optimized for:

- low latency
- fast responses
- document-based question answering
- API integration with Python

Documentation:  
https://ai.google.dev/

---

# Rationale for Model Selection

The Gemini Flash model was chosen for several reasons:

1. **FIU gives us students Gemini Flash for free**

2. **Document understanding**  
   Gemini Flash is optimized for fast responses, making it well suited for interactive applications like chatbots. The API supports file uploads, which allows the model to reference documents directly when generating responses.

3. **Easy Python integration**  
   The Google GenAI SDK provides a clean Python interface, which simplifies the integration with Streamlit.

4. **Scalability**  
   The same architecture can later be expanded into a full RAG system using vector databases or embeddings.

# How the Application Works

1. User uploads a document (PDF or TXT)
2. The file is uploaded to the Gemini API
3. A chat session is initialized with the model
4. The system instruction restricts the AI to only use the uploaded document
5. The user asks questions
6. The AI generates responses grounded in the document

# Ethical AI Reflection (Responsible AI Use)

Large language models such as Gemini provide powerful capabilities for natural language understanding and generation. However, their use introduces several ethical considerations in education. One important concern is that students could begin relying entirely on AI systems to complete assignments without actually understanding the subject matter. When students simply copy answers generated by AI and skip the process of problem-solving or critical thinking, the technology becomes harmful to learning rather than helpful.

Ideally, AI tools should be used as tools for thought, not as replacements for learning. Students can use AI models as assistants that help explain complex ideas, clarify difficult theories, or present concepts in different ways. Because AI can personalize explanations, it can support deeper understanding when used responsibly. In this way, AI should function as a learning companion that helps students think more clearly rather than think less.
